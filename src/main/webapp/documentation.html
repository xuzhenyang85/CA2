How to use GUI 2

  1. Open the Download GUI client 2
  2. Type in the IP-address you want to connects to
  3. In the login-field type: LOGIN:username
  4. Click connect
  5. Write to ALL: In the message field type: MSG:*:message
  6. Write to one client In the message field type: MSG:username:message
  7. Write to multiple clients: In the message field type: MSG:username,username,username:message
  8. The clientlist will be visible for all users with the new logged on clients
  9. Leave the chat: In the message field type: LOGOUT:
  10. The clientlist will be visible for all users with the active users

The design of the system

Klient

Vi har bygget klienten ud fra et interface for at undgå blocking.

Den består af tre dele: GUI(JFrame), Client class, interface

IDataReady interface: observer som hjælper GUI'en med at observere om der er ændringer i client classen (som kommer fra serveren)

Client classen extends Thread, og kører uafhængigt af GUI'en for ikke at blokerer den

Derudover håndterer client class forbindelsen (socket) samt behandler input fra severen

Server

Serveren er opbygget i 2 dele: en server class file(main), client handler som også extends Threads for at flere klienter kan kører på samme tid

Server class file bruger en synchronized list til at håndtere class handler objekterne (de forskellige forbindelser)

Hver gang der oprettes en ny socket(ny forbindelse fra klienten), oprettes der også en ny tråd.

Dette er ikke hensigtsmæssigt i forhold til større chatprogrammer, men for denne opgave virker det fint.

Man kan løse dette med Thread Pools, som bruges til at håndtere mange threads af gangen
